## Observed cap

The largest size at which the END sentinel is preserved verbatim **inline in the prompt context**: **8 KB**.

Above 8 KB, Claude Code does NOT silently truncate. Instead it switches to a "persisted-output" overflow mode: the full hook stdout is written to a file under `~/.claude/projects/<project>/<session>/tool-results/hook-<uuid>-stdout.txt`, and only a ~2KB prefix preview is injected inline, wrapped in a `<persisted-output>` system-reminder pointing to the file. The inline preview consistently cuts off at **POS-00024** (~24 × 80 = ~1920 bytes of POS lines, ~2KB total inline including the BEGIN sentinel) regardless of source size (observed identically at 64 KB and 256 KB).

The full payload is recoverable by Read-ing the persisted file. No hard errors, no crashes, no refusals.

no hard error — overflow is graceful via the persisted-output system-reminder

## Decision

Pick the line that matches the trial:

- [ ] **All sizes ≤ 8 KB pass cleanly.** Design's 8 KB hook budget is safe. Proceed to Week 1 (`PROTOCOL.md` draft) on the existing plan.
- [x] **8 KB passes but 64 KB / 256 KB silently truncates.** Document the observed cap in `PROTOCOL.md` and keep the 8 KB budget as a safety margin.

  Rationale: The 1 KB and 8 KB trials show full inline injection — BEGIN + END sentinels and all POS lines (12 and 102 respectively) appear verbatim in Claude's prompt context. The 64 KB and 256 KB trials behave identically: Claude Code reports `Output too large (64.1KB / 256KB)`, persists the complete stdout to `~/.claude/projects/.../tool-results/hook-*.txt` (verified intact via `wc`/`head`/`tail` — 64 KB file ends at POS-00819, 256 KB file ends at POS-03276 with END sentinel), and injects only a ~2KB inline preview cutting off at POS-00024 in both cases plus a `<persisted-output>` system-reminder pointing to the file. The inline cap therefore lies between 8 KB (passes cleanly) and 64 KB (overflows), and the cap behaviour is *graceful overflow*, not silent truncation: nothing is lost, but the design's assumption that hook payload arrives verbatim in the prompt window is violated above the cap. v0.1 should keep per-fire payload ≤ 8 KB; for any larger context the agent must Read the persisted file (or use Plan B / MCP). The chosen "silently truncates" branch is the closest fit in the existing checkbox list — strictly speaking the truncation is *announced* via the `<persisted-output>` system-reminder, but the operational implication for the 8 KB hook budget is identical.
- [ ] **8 KB passes but 64 KB / 256 KB causes hard refusal or crash.** Lower the hook budget to a comfortable margin (e.g. 4 KB) and harden `cc-connect doctor` to warn if a single chat Message would exceed it.
- [ ] **8 KB silently truncates.** Lower hook budget to whatever passed cleanly, update PROTOCOL.md, proceed.
- [ ] **Even 1 KB doesn't appear in Claude's context.** Trigger **Plan B**: hook contract is unsuitable. Switch v0.1 plan to MCP-server-as-context-source. Stop; reopen the design doc to revise.
